Ryan Goh (3K)'s Documentation

  1. Planning Phrase

    The planning phrase had definitely helped me much in this project. It had helped me write my thoughts of the initial design out on paper, and also created a reference point for me to refer back to when I have to style something. A huge example could be the design of the home page. It consists of many elements coming together to form a nice-looking home page, such as navigation bars, picture titles, even a calandar. With so much information to include, one would be overwhelmed to create and style the elements nicely in such a short period of time without any planning. However, with the wireframes planning sheet, one can then easily follow the plans to style the page accordingly, with clear picture of what the page should look like. In the end, I had mostly followed the planning in wireframes.txt, and slowly design each part of the page in a top-down manner as per the wireframe planning sheet. It made the task so much less daunting and easier to manage. Moreover, the site looked mostly like how I had envisioned it (in the planning stage) to be!

  2. Overcoming Challenges

    I had definitely faced many challenges when doing this project. When I just started, I was quite new to CSS and HTML, so I find it very difficult to style the pages. In fact, when I started, I found myself stuck with styling just the navigation bar! The navigation links just "refused" to right justify. With the help from Google and my teacher, I managed to overcome this problem through realising that a flexbox with "justify-content: space-between;" would space the two divisions (one that I intend to left-justify, one that I intend to right justify) furthest apart, which automatically justifies the content into their right positions as in the wireframe. As I went on with the project, I tried to use Google to search on many tips and tricks, for example, how to style an embed Google Calendar such that it fits screen size, how to use media queries to conditionally reorder elements in order to fit screen size, how to make a collapsable header, even how to full-justify this text! As I realised, using Google is very effective in solving some simple style issues, as many people would have probably faced, asked and solved these issues, in which I can take inspiration from. Little by little, I managed to overcome all of the small tasks that I set for myself in order to finish this whole website. For me, the challenge that I feel the most ecstatic after solving it, is the problematic alignment of the table of image cards after collapsing the headers in the resources page. Firstly, I wanted the number of columns to change as the page resizes. This is not easily done with the table tag. I tried to search for resources, but none of the resources available can help in changing the number of columns of a table in accordance to viewport width. In the end, I resorted to Javascript to manually calculate the number of columns that can be displayed with the viewport width. It worked - but I faced another problem. The height of the image cards are not the same! Not only does it look quite messy, but it also causes "holes" in the table of image cards after resizing which is very unsightly. Yet again, Google does not realy help; so in the end I used Javascript (again) to calculate the maximum height of the image cards and then set the image cards to the same maximum width. I was so relieved and felt so accomplished when it finally worked, and the image cards styled perfectly aligned.

  3. Project Special Mention

    The one piece of my project that I am most proud about is being able to dynamically load content from csv and relevant html files for my blogs page. Due to the nature of blogs and logs, I would have to implement an easy way for users to add their own blogs and platoon points logs to my website. As implementing a working database system would be overly complex, I decided to use the simpler method of using csv files to store the data. It was still a very difficult task. Firstly, I would have to know how to read files using Javascript. However, it is not as simple as it looks. As it turned out, I would need to send a "GET" request to the browser to load the file, wait for it to respond, before I can read the file. Fortunately, using Google, this issue is solved quickly. Next, I would need to display the html files that I get from the csv file. This involves a lot of creation of new elements (div to hold the html file, title etc.), just so that the blog looks relatively nice (bolded title, responsive images etc.) while still having the essential content. Next, since my site has pages for blogs, we need to implement a page system where we show 3 blogs at one time, and then the user can click the left/right arrow buttons to see more. This involves even more Javascript as we now have to save the current index of the first blog shown, and then advance this variable to point to the right current first blog when the user clicks the left or right arrow button, so that older / newer blogs can be shown. Last but not least, we need to implement functionality so that the user can filter out blogs with the month / year that the user wants. To create the sorted navigation list, I have to retrieve the dates from the csv, make sure that all the year/month pair are unique, sort the year/month pairs in chronological order, then display the lists with workable links that actually filter the blogs. Sorting dates was not easy, but after using suitable data structures such as sets, converting to Dates from strings etc, the list was finally done. And with more (painful) scripting that allows filtering of data that contains a specific year or month/year pair, the filter functionality also worked! For sure, that was the most tiring functionality to be added. However, it was all worth it when it succeeded! In the end, I wish that this project would be a stepping stone to help Red cross organise information to our cadets better, and also create a platform where our seniors can communicate (through blogs) to our juniors in a more light-hearted way. I wish that this project would help revive the old RIRC Blogs page, and add additonal functionality (such as platoon points log sheet) to further reassure cadets that they can find their information all in this webpage.
